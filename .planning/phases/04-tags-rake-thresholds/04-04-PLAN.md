<objective>
Deliver the final two tag capabilities: (1) bulk end-by-tag operation per TAG-02 — POST /api/v1/tags/{tag}/end atomically settles every ACTIVE stream carrying that tag in a single DB transaction, with idempotency, silent skip of already-settled streams, and one StreamSettledEvent per settled stream so AFTER_COMMIT cleanup runs correctly; (2) tag TTL cleanup job per TAG-05 — a ShedLock-guarded @Scheduled job that deletes tag_committed_totals rows past the configured inactivity period.

Purpose: Bulk end-by-tag is the core platform-side operation when a session/group lifecycle ends (e.g., "stop all streams in this room"). The TTL cleanup prevents the tag totals table from growing unbounded — TAG-04's aggregate query is only useful while the tag has recent activity, and stale rows pollute storage.
