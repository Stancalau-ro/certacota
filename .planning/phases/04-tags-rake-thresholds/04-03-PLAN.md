<objective>
Deliver streaming rake settlement vertical slice: stopping a rake-enabled streaming transaction performs the same atomic three-way debit/credit/credit split that TransactionServiceImpl.transfer() implements for discrete transfers, with the new arithmetic enforced by a DB check constraint per RAKE-04. Auto-termination uses the same path so a balance-exhausted rake stream still produces correct rake-side credits per D-12. Covers RAKE-02, RAKE-03, RAKE-04.

Purpose: Streaming rake unblocks the marketplace-style use case where each second of streamed tokens routes a configurable share to a platform account (e.g., transcode service taking 10% per second of viewer→creator stream). Phase 3 already laid the lock-ordering pattern in TransactionServiceImpl.transfer; this plan applies the exact same arithmetic and lock order inside stopStream(). The DB check constraint is the safety net — RAKE-04 ensures arithmetic correctness even if the service code is later refactored.

Output: V12 migration adding 5 new columns + check constraint to streaming_transactions, StreamingServiceImpl.stopStream extension to acquire to-account/platform-account locks and perform three-way split (also feeding tag_committed_totals.total_credited_recipient with the correct toAccountAmount = settled - rake), extended StreamingSteps for rake step sentences, all four streaming-rake.feature scenarios GREEN.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Service code -> DB check constraint | Service-layer arithmetic must produce balanced rows; DB constraint is the safety net per RAKE-04 |
| Concurrent stopStream calls -> account locks | Multiple streams settling simultaneously must acquire account locks in consistent order to prevent deadlock |
| Redis stream hash -> stopStream rake field read | Stale or partial Redis hash data must fall back to Postgres per D-13 (consistent with Phase 3 D-17 pattern) |

Bypassing the check would require executing a raw SQL UPDATE bypassing JPA — out of scope for service code.
