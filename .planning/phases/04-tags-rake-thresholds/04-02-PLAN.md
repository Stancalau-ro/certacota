<objective>
Deliver vertical slice: a user can start a tagged streaming or discrete transaction, the engine maintains tag_committed_totals atomically with each settlement, and `GET /api/v1/tags/{tag}/aggregate` returns the committed total from Postgres plus the in-flight projection from Redis with derived rake totals per D-07. Covers TAG-01, TAG-03, TAG-04, TAG-06.

Purpose: Tags are the new grouping abstraction (replacing the deferred session lifecycle). This plan ships a complete observable user-facing capability — caller can tag any transaction and immediately query a real-time aggregate that combines durable Postgres state with live Redis projections. Subsequent plans add streaming rake (Plan 03) and bulk end-by-tag (Plan 04) but do NOT block this aggregate query from working.

Output: V11 schema (3 new tables), TagCommittedTotals entity + repository, tags field on StartStreamRequest/CreditRequest/DebitRequest/PostTransferRequest with @Size validation, StreamState extension, RedisStreamRegistry tag-set support, StreamSettledEvent extension with tags, StreamingServiceImpl.stopStream extended to update tag totals, TransactionServiceImpl credit/debit/transfer extended to write discrete_transaction_tags + update totals, TagService port + TagServiceImpl.aggregate implementation, TagController, TagAutoConfiguration registered in AutoConfiguration.imports, TagSteps wired against the live API.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request -> tag-carrying DTO | Caller-supplied tag strings cross from untrusted client into the engine; unbounded length / count is a DoS vector |
| Service -> Postgres tag_committed_totals | Concurrent settlements on the same tag race for the row lock; non-deterministic lock order causes deadlock |
| Service -> Redis tag-streams Sets | A failed Redis SADD/SREM during register/remove leaves Postgres and Redis state divergent until startup reconciliation |
| AFTER_COMMIT event handler -> Redis SREM | If the AFTER_COMMIT handler crashes between Postgres commit and Redis SREM, tag-streams Set retains a stale streamId until next startup reconciliation |
