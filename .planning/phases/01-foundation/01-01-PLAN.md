## Spring Boot Version Notice

Decision D-02 specified Spring Boot 3.4.x. RESEARCH.md confirms that Spring Boot 3.4 OSS support ended December 31, 2025. This plan uses **Spring Boot 3.5.3** (current OSS-supported release, drop-in compatible). All BOM and plugin versions in the Gradle files below reflect 3.5.3.

## Critical Pitfalls (must not occur)

1. DO NOT apply `org.springframework.boot` plugin to `engine-core` or `engine-spring` — it creates a fat JAR that breaks the module dependency graph.
2. BOTH `flyway-core` AND `flyway-database-postgresql` must be declared in `engine-service/build.gradle` — Flyway 10+ split Postgres support into a separate artifact. Omitting `flyway-database-postgresql` causes startup failure: "No database found to handle jdbc:postgresql:...".
3. DO NOT use `@RunWith(Cucumber.class)` — this is the JUnit 4 runner, deprecated since Cucumber 7. Use `@Suite` + `@IncludeEngines("cucumber")`.
4. DO NOT place `spring.datasource.url` with a hardcoded localhost in `application.yml` or `application-test.yml` — `@ServiceConnection` overrides datasource at the Spring context level; a hardcoded URL fights it.
5. EXACTLY ONE class in the glue path carries `@CucumberContextConfiguration`. Do not annotate step definition classes with it.
</context>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build classpath | Gradle resolves dependencies from Maven Central — supply chain boundary |
| Test container | Testcontainers spins up Postgres inside Docker — network boundary between JVM and container |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Flyway migrations (V1/V2/V3) | mitigate | Flyway checksums validate migration files on every startup; any modification to a committed migration causes startup failure — this is the intended behavior |
| T-01-02 | Spoofing | Testcontainers Postgres credentials | accept | Test-only container with auto-generated credentials; never exposed outside the test JVM; no production data |
| T-01-03 | Elevation of Privilege | Maven Central supply chain | accept | Standard risk for Spring Boot projects; BOM pins versions; no snapshot or non-release dependencies used |
</threat_model>
