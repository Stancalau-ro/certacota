<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Redisson consumer thread → StreamingServiceImpl.autoTerminate | Internally-triggered settlement; stream state from Redis used as-is |
| FallbackSweepJob → Postgres streaming_transactions | Read-only query on ACTIVE rows; auto-terminates any at-floor stream |
| AuditArchivalJob → public.balance_audit_log + audit_archive schema | JDBC batch write+delete; must not partially execute without ShedLock |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-13 | Tampering | Auto-termination double-settle | mitigate | autoTerminate() checks streaming_transactions.status == ACTIVE before acquiring account lock; if SETTLED already, returns immediately (Pitfall 5 idempotency guard) |
| T-3-14 | Denial of Service | FallbackSweepJob running on multiple pods simultaneously | mitigate | @SchedulerLock(name="streaming_fallback_sweep") with ShedLock Postgres lock; LockAssert.assertLocked() guards body execution |
| T-3-15 | Tampering | AuditArchivalJob deleting audit rows without archiving | mitigate | INSERT INTO audit_archive before DELETE enforced in method body order; DELETE in same JDBC transaction context; D-34 invariant |
| T-3-16 | Denial of Service | AuditArchivalJob running for >lockAtMostFor on pod crash | mitigate | lockAtMostFor="PT2H" releases lock automatically if pod crashes during long archival |
| T-3-17 | Tampering | TransactionServiceImpl.debit — floor bypass when Redis fails | mitigate | if Redis fails and ACTIVE rows exist in Postgres, throw RedisUnavailableException (503) per D-30; credits always allowed regardless |
| T-3-18 | Tampering | AccountServiceImpl.closeAccount — active stream bypass when Redis fails | mitigate | if Redis fails and ACTIVE rows in Postgres, throw RedisUnavailableException (503) per D-32; close blocked until Redis recovers |
</threat_model>
