## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-06 | Tampering | startStream — CLOSED account guard | mitigate | AccountStatus.CLOSED check after findWithLock() before any write (per D-20); returns 409 AccountClosedException |
| T-3-07 | Tampering | startStream — double-start same streamId | mitigate | streamRegistry.get(streamId) + streamingTransactionRepository.findByStreamId check inside lock; throw StreamAlreadyActiveException (409); UNIQUE(stream_id) DB constraint is last-resort guard |
| T-3-08 | Tampering | stopStream settlement — floor violation via concurrent debits | mitigate | clampToAvailableBalance() applies min(projected, available-floor) per D-19; account never goes below floor; no exception thrown to leave stream in ACTIVE state permanently |
| T-3-09 | Tampering | startStream — zero/negative ratePerSecond | mitigate | @NotNull @Positive on ratePerSecond in StartStreamRequest; MethodArgumentNotValidException → 400 via existing GlobalExceptionHandler |
| T-3-10 | Information Disclosure | estimateBalance — reads committed balance | accept | Engine trusts caller-supplied accountId; authentication/authorization is platform's responsibility (REQUIREMENTS.md out of scope) |
| T-3-11 | Denial of Service | Redis unavailable — streaming paths return 503 | mitigate | RedisUnavailableException thrown from RedisStreamRegistry on RedisConnectionFailureException; GlobalExceptionHandler maps to 503; discrete credits always allowed per D-31 |
