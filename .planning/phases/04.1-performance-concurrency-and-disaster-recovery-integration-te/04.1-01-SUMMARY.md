### LatencyRecorder

**File:** `engine-service/src/test/java/com/certacota/engine/service/perf/LatencyRecorder.java`

Thread-safe via `ConcurrentLinkedQueue<Long>`. Each thread calls `record(elapsedMs)`. After all threads finish, call `snapshot()` to get p50/p95/p99/max across all collected values. Clamps negative values to 0.

```java
LatencyRecorder recorder = new LatencyRecorder();
// In each thread: long start = System.currentTimeMillis(); ... recorder.record(System.currentTimeMillis() - start);
LatencyRecorder.Snapshot snap = recorder.snapshot();
// snap.p50Ms(), snap.p95Ms(), snap.p99Ms(), snap.maxMs(), snap.count()
```

The nested `record Snapshot(long p50Ms, long p95Ms, long p99Ms, long maxMs, long count)` is Jackson-serializable without any additional configuration.

### DrContainerHolder

**File:** `engine-service/src/test/java/com/certacota/engine/service/DrContainerHolder.java`

`@TestConfiguration(proxyBeanMethods = false)` class with two static containers started in a static block. DR test classes import this instead of `TestcontainersConfiguration`.

```java
// DR test class annotation:
@SpringBootTest(webEnvironment = RANDOM_PORT)
@Import(DrContainerHolder.class)   // NOT TestcontainersConfiguration
@ActiveProfiles({"test", "perf-test"})
@Tag("disaster-recovery")
class PostgresOutageRecoveryIT {

    @Test
    void pauseAndUnpause() throws Exception {
        DockerClient client = DockerClientFactory.lazyClient();
        String containerId = DrContainerHolder.getPostgresContainer().getContainerId();
        client.pauseContainerCmd(containerId).exec();
        // ... test behavior ...
        client.unpauseContainerCmd(containerId).exec();
        Thread.sleep(500); // allow HikariCP reconnect

        String redisId = DrContainerHolder.getRedisContainer().getContainerId();
        client.stopContainerCmd(redisId).exec();
        client.startContainerCmd(redisId).exec();
    }
}
```

`TestcontainersConfiguration.java` is byte-identical to its pre-plan state. All non-DR test classes keep `@Import(TestcontainersConfiguration.class)`.

### perf-db Docker Image

**Dockerfile:** `engine-service/src/test/docker/perf-db/Dockerfile`

Build args `ACCOUNTS` (default 1000) and `TXN_PER_ACCOUNT` (default 100) are propagated as `ENV` variables so `seed.sh` can access them at container init time.

Image name convention: `certacota-perf-db-a{ACCOUNTS}-t{TXN_PER_ACCOUNT}` (assigned by the caller of `docker build -t`).

**seed.sh:** `engine-service/src/test/docker/perf-db/seed.sh`

Creates and populates two tables via `generate_series()` at container init time:

| Table | Schema | Row count |
|-------|--------|-----------|
| `stress_accounts` | `id VARCHAR(36) PK, balance NUMERIC(36,18), created_at TIMESTAMPTZ` | `${ACCOUNTS}` |
| `stress_audit_log` | `id BIGSERIAL PK, account_id VARCHAR(36), amount NUMERIC(36,18), recorded_at TIMESTAMPTZ` | `${ACCOUNTS} * ${TXN_PER_ACCOUNT}` |

Account ID format: `'stress-acc-' || LPAD(n::text, 6, '0')` → `stress-acc-000001` through `stress-acc-{ACCOUNTS}`

Initial balance: `1000000.000000000000000000` (scale=18, matches Plan 05's conservation invariant SUM)

Audit log account distribution: round-robin via `((n - 1) % ${ACCOUNTS} + 1)` — every account gets exactly `TXN_PER_ACCOUNT` audit rows.

Index: `CREATE INDEX IF NOT EXISTS idx_stress_audit_account ON stress_audit_log (account_id)` — required for latency assertions in LargeDbStressIT.

**Usage in LargeDbStressIT (Plan 05):**
```java
String imageName = String.format("certacota-perf-db-a%d-t%d",
    config.getStress().getAccounts(), config.getStress().getTransactionsPerAccount());
ensureSeedImageExists(imageName, config.getStress().getAccounts(), config.getStress().getTransactionsPerAccount());
// Start container and wire via @DynamicPropertySource (see RESEARCH Pattern 6)
```

## Deviations from Plan

### Auto-added: .gitattributes for LF enforcement

**Rule 2 (Missing critical functionality)**
- **Found during:** Task 3
- **Issue:** Writing seed.sh via the Write tool on Windows triggers git's CRLF-on-checkout behavior. Without a `.gitattributes` rule, anyone cloning on Windows would get `seed.sh` with CRLF line endings, which breaks the `#!/bin/sh` shebang and `set -e` inside a Linux Docker container.
- **Fix:** Added `.gitattributes` with `engine-service/src/test/docker/**/*.sh text eol=lf` to force LF on all Docker shell scripts regardless of checkout platform.
- **Files modified:** `.gitattributes` (new file)
- **Commit:** a5298e1

### Deviation: LatencyRecorder uses ConcurrentLinkedQueue, not HdrHistogram

The plan's `must_haves` section and task action both specify `ConcurrentLinkedQueue<Long>` as the implementation (explicitly noted: "RESEARCH Pattern 5 recommendation — avoids HdrHistogram thread-safety issue"). Implemented accordingly. HdrHistogram remains available on the classpath as a fallback if Plans 02-05 prefer it, but LatencyRecorder does not depend on it.

### Deviation: IntervalRange and AmountRange as named top-level static classes in PerfTestProperties

The plan action specified `IntervalRange` and `AmountRange` as field types inside `Mixed` but did not specify their exact nesting level. Made them top-level static inner classes of `PerfTestProperties` (not nested inside `Mixed`) so they can be reused by future plans without fully-qualified inner class references. The Java field declarations `IntervalRange creditIntervalMs`, `AmountRange creditAmount`, etc. are present as specified.
