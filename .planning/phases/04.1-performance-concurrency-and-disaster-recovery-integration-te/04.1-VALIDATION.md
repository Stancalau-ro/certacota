# Phase 04.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | JUnit 5 + Spring Boot Test + Testcontainers |
| **Config file** | `engine-service/build.gradle` (`performanceTest` + `disasterRecoveryTest` tasks; tag filters on default `test`) |
| **Quick run command** | `./gradlew :engine-service:performanceTest --tests "*.ConcurrentSessionsIT"` |
| **Full suite command** | `./gradlew :engine-service:performanceTest && ./gradlew :engine-service:disasterRecoveryTest` |
| **DR suite command** | `./gradlew :engine-service:disasterRecoveryTest` |
| **Estimated runtime** | ~30–120 seconds per IT (varies by concurrency config); full suite ~5–10 minutes |

> **Tag-filter discipline:** Phase 04.1 tests are excluded from the default `./gradlew :engine-service:test` task (`excludeTags 'disaster-recovery', 'performance', 'concurrency'`). They run ONLY under `performanceTest` (tags `performance | concurrency`) or `disasterRecoveryTest` (tag `disaster-recovery`). Per-task verify commands MUST target the matching Gradle task — using `:test --tests` against an excluded class produces 0 tests executed and a false-green exit 0.

---

## Sampling Rate

- **After every task commit:** Run the IT's matching Gradle task (`performanceTest` or `disasterRecoveryTest`) scoped via `--tests` to just the new/touched IT class
- **After every plan wave:** Run the full task for that wave (entire `performanceTest` or `disasterRecoveryTest`)
- **Before `/gsd-verify-work`:** Full suite (`performanceTest` + `disasterRecoveryTest`) must pass
- **Max feedback latency:** 120 seconds (configurable via `concurrency.sessions.count`)
