The script is the single source of truth for the seeded test dataset and is invoked once at container init time by the Postgres entrypoint via `/docker-entrypoint-initdb.d/01-seed.sh`. ACCOUNTS and TXN_PER_ACCOUNT are read from the environment (set in the Dockerfile via `ENV` from `ARG`).

    File body (exact contents — `#!/bin/sh` shebang, `set -e`, then a `psql` heredoc):
    ```
    #!/bin/sh
    set -e

    echo "perf-db seed.sh starting: ACCOUNTS=${ACCOUNTS}, TXN_PER_ACCOUNT=${TXN_PER_ACCOUNT}"

    psql -v ON_ERROR_STOP=1 --username "$POSTGRES_USER" --dbname "$POSTGRES_DB" <<-EOSQL
        CREATE TABLE IF NOT EXISTS stress_accounts (
            id VARCHAR(36) PRIMARY KEY,
            balance NUMERIC(36,18) NOT NULL,
            created_at TIMESTAMPTZ DEFAULT NOW()
        );

        INSERT INTO stress_accounts (id, balance, created_at)
        SELECT
            'stress-acc-' || LPAD(n::text, 6, '0'),
            1000000.000000000000000000,
            NOW()
        FROM generate_series(1, ${ACCOUNTS}) AS n;

        CREATE TABLE IF NOT EXISTS stress_audit_log (
            id BIGSERIAL PRIMARY KEY,
            account_id VARCHAR(36) NOT NULL,
            amount NUMERIC(36,18) NOT NULL,
            recorded_at TIMESTAMPTZ DEFAULT NOW()
        );

        INSERT INTO stress_audit_log (account_id, amount, recorded_at)
        SELECT
            'stress-acc-' || LPAD(((n - 1) % ${ACCOUNTS} + 1)::text, 6, '0'),
            1.000000000000000000,
            NOW()
        FROM generate_series(1, ${ACCOUNTS} * ${TXN_PER_ACCOUNT}) AS n;

        CREATE INDEX IF NOT EXISTS idx_stress_audit_account ON stress_audit_log (account_id);
    EOSQL

    echo "perf-db seed.sh completed: seeded ${ACCOUNTS} accounts and $((ACCOUNTS * TXN_PER_ACCOUNT)) audit-log rows"
    ```

    Notes the executor must follow:
    - Use `${ACCOUNTS}` and `${TXN_PER_ACCOUNT}` shell substitution OUTSIDE the heredoc terminator's quoting — the heredoc opener is `<<-EOSQL` (not `<<-'EOSQL'`) so the shell expands the variables before psql receives the SQL. This is required by RESEARCH Pitfall 6 (do NOT use psql `:'ARG'` substitution).
    - The account ID format is `stress-acc-` followed by zero-padded 6-digit number (LPAD), matching what `LargeDbStressIT` (Plan 05) probes via REST: `"stress-acc-" + String.format("%06d", i)`.
    - Initial balance is exactly `1000000.000000000000000000` at scale=18 — Plan 05 multiplies this by the row count to compute the initial token total for the conservation invariant (D-20).
