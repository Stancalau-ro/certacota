<objective>
Build `MixedConcurrencyIT` — the per-account organic simulation that exercises hot-account contention patterns (D-11 + D-12). Each account in the test pool has an independent event lifecycle running for `durationSeconds`, generating streaming starts, discrete credits, and discrete debits at random intervals. Hot accounts receive `contentionMultiplier`× more events than average accounts, surfacing lock-wait starvation and deadlock risk under asymmetric load.

Isolated to its own plan because the per-account event-runner model is heavier than the simpler latch-based tests in Plan 02 — keeps each executor context manageable.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test JVM → engine-service REST API | All test traffic crosses the public HTTP boundary; per-account event runner uses RestTemplate only |
| Per-account event-runner threads → shared `AtomicReference<BigDecimal>` accumulators | Multiple runners concurrently update `totalCredited`, `totalDebited`, `totalSettled`; updateAndGet is the only safe mutation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04.1-08 | Tampering | Concurrent BigDecimal accumulation across `totalAccounts` threads | mitigate | All shared BigDecimal mutations go through `AtomicReference.updateAndGet(curr -> curr.add(delta))` — CAS-based, no read-then-write races |
| T-04.1-09 | Denial of Service | Per-account threads could deadlock waiting on locked accounts under hot-account contention | mitigate | `done.await(durationSeconds + 60, TimeUnit.SECONDS)` is the deadlock canary — if any runner is wedged, await returns false and the assertion fails. Hot-account multiplier is bounded by config (default 3.0, tunable) |
| T-04.1-10 | Information Disclosure | JSON report captures account IDs and balances | accept | Synthetic test data only (`perf-mixed-acc-0000` etc.). Reports gitignored |

**No new production attack surface introduced — exercises Phase 2 + Phase 4 REST endpoints only.** ASVS L1 baseline.
</threat_model>
